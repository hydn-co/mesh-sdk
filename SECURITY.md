# Security Policy

Hydden Co welcomes responsible vulnerability reports. This repository is public but proprietary; please follow the guidelines below when reporting security issues.

## Reporting a Vulnerability
- Do NOT open issues containing security-sensitive information in this public repository.
- Send an email to security@hydden.com with:
  - A concise description of the issue and a reproducible test case (if possible).
  - Affects: which component or file.
  - Severity assessment and impact.
  - Your contact information and disclosure preferences.

## Response Process
- We will acknowledge receipt within 72 hours.
- We will coordinate remediation and public disclosure timelines where appropriate.

## Public Disclosure
We follow coordinated disclosure practices. Unless otherwise agreed, please do not publicly disclose vulnerabilities before we have had an opportunity to fix them.

## Security Best Practices for Contributors
- Do not commit secrets, private keys, or credentials.
- Use environment variables or secret managers for sensitive values.
- Run `gitleaks` or similar scan locally before pushing changes.
